Project: LAN Gateway Node

LAN Gateway Node is a single Raspberry Pi Pico + W5500 Ethernet project.

The Pico becomes a small LAN appliance that:

- starts automatically from root main.py
- serves a browser UI
- creates an admin account on first launch
- requires login for protected routes
- generates runtime session, client, and internal backend tokens
- expires sessions after a configured inactivity timeout
- protects setup and login with CSRF tokens
- locks out repeated failed logins
- writes a rotating persistent audit log
- validates config before starting network service
- exposes health and safe status export endpoints
- displays its current DHCP IP address
- serves protected local backend data
- records recent requests
- rate-limits clients
- exposes metrics
- keeps UDP discovery disabled by default

Network shape:

LAN client browser
    |
    | HTTP request
    v
Raspberry Pi Pico + W5500
    |
    | in-process protected calls
    v
local_backend.py

This is not an inline firewall. It is an authenticated application-layer LAN service running on constrained hardware.

Important endpoints:

/setup       first-run admin password setup
/login       admin login
/logout      clear auth cookies
/            dashboard
/metrics     metrics and request log
/backend     local backend summary
/status      local backend status
/api         local backend API data
/health      JSON health and diagnostics
/export      safe status export
/test/start  small in-process load test
/admin       intentionally blocked

Security design:

- Passwords are submitted with POST.
- A password salt, repeated password hash, and hash round count are stored in settings.py on the Pico.
- Runtime bearer tokens are held in memory and validated by hash.
- settings.py is ignored by git.
- Protected routes require a valid session cookie and client token cookie.
- A backend token is used as an internal boundary between the gateway route layer and local_backend.py.
- Inactive expired sessions are cleared and must log in again.
- Failed login attempts are tracked and temporarily locked out.
- audit.log keeps a small rotating persistent record of recent requests.
- /export includes safe diagnostics but no passwords or bearer tokens.
- UDP discovery is off unless DISCOVERY_ENABLED is set to True.
- Recent request logs and simple rate limits are enabled.

Limits:

- The Pico does not provide production HTTPS serving in this project.
- Use a TLS reverse proxy, VPN, or router HTTPS feature for encrypted access.
- A standard Pico does not provide secure boot or flash encryption.
- Anyone with physical access to USB data, BOOTSEL, reset, or SWD should be treated as able to compromise the device.
- Router firewall rules and VLANs must be configured on the router or switch, not in Pico firmware.

Best presentation:

This is a realistic embedded LAN service demo. It shows Ethernet bring-up, socket handling, authenticated browser routes, token-backed access control, request diagnostics, and honest security boundaries on constrained hardware.
